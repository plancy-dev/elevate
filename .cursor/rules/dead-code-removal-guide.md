# Dead Code 제거 가이드라인

**기반 원칙**: 로버트 C. 마틴의 클린 코드 & 보이스카우트 원칙  
**목표**: 사용하지 않는 코드를 안전하게 제거하여 코드베이스 유지보수성 향상

---

## 🎯 원칙

### 1. 확실한 것만 제거

> "의심스러우면 유지하라. 확실하지 않은 코드를 제거하는 것은 위험하다."

**적용 방법**:
- ts-prune으로 확인된 것만 제거
- 동적 import 가능성 확인
- 타입 정의 파일은 제거하지 않음

### 2. 작은 단계로 진행

> "리팩터링은 작은 단계로 진행해야 한다."

**적용 방법**:
- 한 번에 하나의 파일/함수만 제거
- 각 단계마다 빌드 확인
- 커밋 단위를 작게 유지

### 3. 검증 후 제거

> "제거 전 반드시 검증하라."

**검증 항목**:
- [ ] ts-prune으로 확인됨
- [ ] import되지 않음
- [ ] 동적 import 없음
- [ ] 빌드 성공
- [ ] 타입 체크 통과

---

## 🔍 분석 도구

### 1. ts-prune

```bash
pnpm run find:deadcode
```

**확인 항목**:
- 사용하지 않는 export
- 사용하지 않는 파일

### 2. ESLint unused-imports

```bash
pnpm run lint:unused
```

**확인 항목**:
- 사용하지 않는 import
- 사용하지 않는 변수

### 3. 수동 확인

**확인 방법**:
```bash
# 특정 파일이 import되는지 확인
grep -r "from.*lib/api" src/
```

## 🚨 제거하지 말아야 할 것

### 1. 동적 import로 사용되는 파일

```typescript
// ❌ 제거하면 안 됨
const module = await import('./module')
```

**확인 방법**:
- `import(` 패턴 검색
- 런타임 로딩 가능성 확인

### 2. 타입 정의 파일

```typescript
// ❌ 제거하면 안 됨
export type User = { id: string; email: string; }
```

**이유**:
- 타입은 컴파일 타임에만 사용됨
- ts-prune이 타입을 감지하지 못할 수 있음

### 3. 테스트 파일

- 테스트는 별도로 관리
- 테스트 파일은 제거하지 않음

### 4. 문서/설정 파일

- README, 설정 파일 등
- 문서는 코드베이스의 일부

## 📋 제거 워크플로우

### Step 1: 분석

1. ts-prune 실행
2. ESLint unused-imports 확인
3. 수동 확인 (필요시)

### Step 2: 영향도 분석

1. 제거 가능한 파일 목록 작성
2. 제거 시 영향받는 파일 확인
3. 제거 후 빌드 영향도 분석

### Step 3: 안전한 제거

1. 확실한 것부터 제거
2. 작은 단계로 진행
3. 각 단계마다 빌드 확인

### Step 4: 검증

1. 빌드 성공 확인
2. 타입 체크 통과 확인
3. 린트 오류 없음 확인
4. 기능 테스트 (가능한 경우)

## ✅ 체크리스트

### 제거 전

- [ ] ts-prune으로 확인됨
- [ ] import되지 않음
- [ ] 동적 import 없음
- [ ] 타입 정의 파일 아님
- [ ] 테스트 파일 아님
- [ ] 문서/설정 파일 아님

### 제거 후

- [ ] 빌드 성공
- [ ] 타입 체크 통과
- [ ] 린트 오류 없음
- [ ] 커밋 메시지 작성
- [ ] 문서 업데이트
